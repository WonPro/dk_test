1.Project setup
:npm install

2. 프로젝트 구동
:npm run serve

3. 문제 해결 과정.
    1) 프레임워크는 Vue.js(3.x)로 제작.

    2) contents를 받아올 dummy data들은 임시의 js파일로 저장하여 받도록 함.

    3) 메인이 될 index파일은 App.vue에 저장, 나머지 에셋들은 components화 시킴.

    4) 헤더
        a. 모바일 구간의 LNB toggle button을 이미지로 만들지 않고 Element로 만듬.

    5) 사이드메뉴
        a. 디자인파일에는 사이드메뉴가 접혀지더라도 콘텐츠 콘테이너의 크기변화는 없으나, 반응형 웹의 특성을 살려 넓혀진 공간에 맞게 리사이징 되도록 함.
        b. 사이드메뉴의 태그별 정렬기능 fliter메소드를 사용하여 만들고자 하였으나 실패
            *추가로 태그명을 클릭했을때 어떤 탭의 콘텐츠가 보여져야 하는지 명확하지 않음
        c. 모바일 영역 시작시 메뉴가 오픈된 상태로 진행

    6) 콘텐츠 영역
        a. '인기 블로그' 탭의 New / Hot 아이콘의 경우 제목이 Ellipsis될 경우 보이지 않는 문제 협의 필요.
        b. '인기 스토리' 탭의 BEST 섹션의 5개 스토리가 어떤 기준으로 보여지는지 정의되지 않아 임의로 5개를 보여주기로 함.
        C. '인기 스토리' 탭의 Carosel이 개발환경인 Vue3용 라이브러리가 나오지 않아 제대로 표현되지 못하였으나, 개발할 시간이 부족했던 관계로 디자인파일과 다르게 표현됨.
    
    7) 푸터
        a. 위치에 대해서 sticky 형식으로 할지, static으로 할 지 고민하였으나 LNB의 스타일을 고려하여 sticky형식으로 진행하였음.

4. 기타 사항
    1) vue.js 3버전에 완벽호환되는 carosel 라이브러리를 찾지 못하여 슬라이드 초반에 보여지는 모습이 완벽하지 못함.

    2) 인터넷익스플로러 지원 종료로 인하여 IE의 호환성검사는 따로 진행하지 않았으나, 11버전에서는 대부분 지원할것이라 예상됨.

    3) 640px이하의 실제 모바일 사이즈에서 어떤 폰트사이즈와 어떤 버튼사이즈를 구성해야 하는지에 대한 정의가 불분명하여 디자인파일의 폰트사이즈를 최대한 구성하였으나 몇몇의 element들은 임의로 변경함.

    4) 포토샵에 지정된 폰트 사이즈는 Pt단위로 Px단위와 조금 다를 수 있으나 실제 퍼블리싱때에는 Px단위로 지정함.
